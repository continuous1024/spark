# Apache Spark 核心类和接口功能说明

## Spark Core 核心类

### SparkContext

- **功能**：Spark功能的主要入口点，表示与Spark集群的连接
- **职责**：
    - 创建RDD、累加器和广播变量
    - 访问Spark服务（如调度器）
    - 设置配置参数
    - 管理作业执行

### RDD (Resilient Distributed Dataset)

- **功能**：弹性分布式数据集，Spark的核心抽象
- **职责**：
    - 表示分布在集群中的不可变数据集合
    - 提供转换和行动操作
    - 跟踪血统信息用于容错
    - 支持分区和数据本地性

### Partitioner

- **功能**：决定RDD中的元素如何分区
- **职责**：
    - 确定每个键应该去哪个分区
    - 支持哈希分区和范围分区

### Scheduler

- **功能**：调度和执行任务
- **职责**：
    - 将逻辑计划转换为物理执行计划
    - 将任务分配给执行器
    - 处理任务失败和重试

### ShuffleManager

- **功能**：管理shuffle操作
- **职责**：
    - 协调数据重分布
    - 优化网络传输
    - 管理中间数据

## Spark SQL 核心类

### SparkSession

- **功能**：Spark 2.0引入的新入口点，用于处理结构化数据
- **职责**：
    - 创建DataFrame和Dataset
    - 执行SQL查询
    - 访问配置
    - 管理缓存表

### DataFrame / Dataset

- **功能**：分布式的命名列集合
- **职责**：
    - 提供结构化数据处理API
    - 支持SQL操作
    - 与外部数据源集成
    - 优化查询执行

### SQLContext

- **功能**：Spark 1.x中的SQL功能入口点（现已被SparkSession取代）
- **职责**：
    - 创建DataFrame
    - 执行SQL查询
    - 注册临时表

### Catalyst

- **功能**：查询优化框架
- **职责**：
    - 解析SQL
    - 生成逻辑计划
    - 应用优化规则
    - 生成物理计划

### DataSource

- **功能**：外部数据源接口
- **职责**：
    - 读写外部数据
    - 提供架构信息
    - 支持谓词下推和列剪裁

## Structured Streaming 核心类

### StreamingQuery

- **功能**：表示一个连续执行的流查询
- **职责**：
    - 管理查询生命周期
    - 提供查询状态和进度信息
    - 支持启动、停止和等待完成操作

### Source

- **功能**：流数据源抽象
- **职责**：
    - 提供连续到达的数据
    - 跟踪处理进度（偏移量）
    - 支持从检查点恢复

### Sink

- **功能**：流数据输出抽象
- **职责**：
    - 将结果写入外部系统
    - 确保输出一致性
    - 支持不同的输出模式

### StreamExecution

- **功能**：流查询执行引擎
- **职责**：
    - 协调源和接收器
    - 管理状态存储
    - 处理触发器和微批次
    - 实现容错机制

### Offset

- **功能**：表示流处理进度
- **职责**：
    - 跟踪每个源的处理位置
    - 支持序列化和反序列化
    - 用于检查点和恢复

## Spark Streaming 核心类

### StreamingContext

- **功能**：Spark Streaming功能的主要入口点
- **职责**：
    - 创建DStream
    - 设置批处理间隔
    - 启动和停止流处理
    - 管理检查点

### DStream (Discretized Stream)

- **功能**：表示连续的RDD序列
- **职责**：
    - 提供高级流处理API
    - 管理RDD的创建和处理
    - 跟踪依赖关系

### Receiver

- **功能**：从外部源接收数据
- **职责**：
    - 连接到外部系统
    - 接收和缓冲数据
    - 将数据存储到Spark中

## MLlib 核心类

### Pipeline

- **功能**：机器学习工作流抽象
- **职责**：
    - 组合多个算法和数据转换
    - 简化模型训练和应用
    - 支持持久化和加载

### Transformer

- **功能**：将一个DataFrame转换为另一个DataFrame
- **职责**：
    - 应用特征转换
    - 应用训练好的模型进行预测

### Estimator

- **功能**：可以拟合DataFrame生成Transformer的算法
- **职责**：
    - 实现机器学习算法
    - 从数据中学习参数
    - 生成模型（Transformer）

### Evaluator

- **功能**：评估模型质量
- **职责**：
    - 计算性能指标
    - 支持模型选择和调优

## GraphX 核心类

### Graph

- **功能**：表示一个有向图
- **职责**：
    - 管理顶点和边集合
    - 提供图操作API
    - 支持图算法

### VertexRDD / EdgeRDD

- **功能**：表示图中的顶点和边
- **职责**：
    - 存储图数据
    - 支持高效的图操作
    - 优化存储和计算
