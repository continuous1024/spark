# Apache Spark 常见代码模式和约定

## 代码组织模式

### 包结构约定

Spark代码库遵循以下包结构约定：

```
org.apache.spark              # 顶级包
├── api                       # 公共API
├── deploy                    # 部署相关
├── executor                  # 执行器实现
├── internal                  # 内部实现
├── launcher                  # 应用程序启动器
├── memory                    # 内存管理
├── network                   # 网络通信
├── rdd                       # RDD实现
├── scheduler                 # 调度器
├── shuffle                   # Shuffle实现
├── sql                       # SQL引擎
│   ├── catalyst             # 查询优化框架
│   ├── execution            # 查询执行
│   ├── expressions          # 表达式
│   ├── sources              # 数据源
│   └── types                # 数据类型
├── storage                   # 存储管理
├── streaming                 # 流处理
│   ├── api                  # 流处理API
│   ├── dstream              # DStream实现
│   ├── receiver             # 接收器
│   └── scheduler            # 流调度器
└── util                      # 工具类
```

### 类命名约定

- **接口**：通常不使用`I`前缀，如`Source`而非`ISource`
- **抽象类**：通常使用`Abstract`前缀，如`AbstractScheduler`
- **特质（Trait）**：通常描述能力，如`Serializable`、`Logging`
- **实现类**：通常直接使用具体名称，如`HashPartitioner`
- **伴生对象**：与类同名，包含工厂方法和常量

### 文件组织

- 每个文件通常只包含一个主要类或特质
- 小的辅助类可以与主类放在同一文件中
- 相关的类可以放在同一个包中
- 测试类放在对应的测试包中，通常以`Suite`结尾

## 编程模式

### 函数式编程模式

Spark大量使用函数式编程模式，特别是在RDD和DataFrame API中：

```scala
// 函数式转换链
val result = rdd
  .map(x => x + 1)
  .filter(x => x % 2 == 0)
  .reduce((x, y) => x + y)

// 高阶函数
def transform[T](data: RDD[T], f: T => T): RDD[T] = {
  data.map(f)
}
```

### 不可变性

Spark倾向于使用不可变数据结构和模式：

```scala
// RDD是不可变的
val rdd1 = sc.parallelize(1 to 10)
val rdd2 = rdd1.map(_ * 2) // 创建新RDD，而不是修改rdd1

// 不可变集合
val map = Map("a" -> 1, "b" -> 2)
val newMap = map + ("c" -> 3) // 创建新Map，而不是修改原Map
```

### 延迟计算

Spark使用延迟计算模式优化执行：

```scala
// 转换操作不会立即执行
val rdd1 = sc.parallelize(1 to 10)
val rdd2 = rdd1.map(_ * 2)
val rdd3 = rdd2.filter(_ > 5)

// 只有在行动操作时才会触发计算
val result = rdd3.collect() // 此时才执行上面的转换链
```

### Builder模式

Spark API中广泛使用Builder模式：

```scala
// SparkSession构建器
val spark = SparkSession.builder()
  .appName("MyApp")
  .master("local[*]")
  .config("spark.some.config.option", "some-value")
  .getOrCreate()

// DataFrame读取器
val df = spark.read
  .format("csv")
  .option("header", "true")
  .option("inferSchema", "true")
  .load("path/to/file.csv")
```

### 工厂方法模式

Spark使用工厂方法创建复杂对象：

```scala
// 通过伴生对象创建RDD
val rdd = sc.textFile("path/to/file.txt")

// 通过工厂方法创建Partitioner
val partitioner = Partitioner.defaultPartitioner(rdd1, rdd2)
```

## 错误处理模式

### 异常处理

Spark使用以下异常处理模式：

```scala
// 使用try-catch处理异常
try {
  // 可能抛出异常的代码
} catch {
  case e: IOException =>
    logWarning("IO error", e)
    // 处理IO异常
  case NonFatal(e) =>
    logError("Unexpected error", e)
    // 处理其他非致命异常
} finally {
  // 清理资源
}
```

### 错误报告

Spark使用日志记录错误，而不是返回错误代码：

```scala
// 使用Logger记录错误
logError("Failed to execute task", exception)

// 使用断言验证条件
assert(condition, "Error message if condition is false")
```

## 并发模式

### Actor模式

Spark早期版本使用Akka实现组件间通信，现在使用自定义RPC框架：

```scala
// 消息传递模式
case class TaskSubmitted(taskId: Long, serializedTask: ByteBuffer)

// 处理消息
def receive: PartialFunction[Any, Unit] = {
  case TaskSubmitted(id, task) =>
    // 处理任务提交
}
```

### 并发集合

Spark使用并发集合管理共享状态：

```scala
// 并发映射
val concurrentMap = new ConcurrentHashMap[String, String]()

// 原子引用
val atomicRef = new AtomicReference[State](INITIALIZED)
```

## 测试模式

### 单元测试

Spark使用ScalaTest和JUnit进行单元测试：

```scala
// ScalaTest测试套件
class MyTestSuite extends SparkFunSuite with BeforeAndAfterAll {

  var sc: SparkContext = _

  override def beforeAll(): Unit = {
    sc = new SparkContext("local", "test")
  }

  override def afterAll(): Unit = {
    if (sc != null) {
      sc.stop()
    }
  }

  test("my test") {
    val rdd = sc.parallelize(1 to 10)
    assert(rdd.count() === 10)
  }
}
```

### 集成测试

Spark使用专门的基类进行集成测试：

```scala
// 集成测试基类
class MyIntegrationTest extends QueryTest with SharedSparkSession {

  test("SQL query") {
    val df = spark.sql("SELECT 1")
    checkAnswer(df, Row(1))
  }
}
```

## 性能优化模式

### 内存管理

Spark使用自定义内存管理避免GC压力：

```scala
// 使用Tungsten内存管理
val memoryManager = new UnifiedMemoryManager(
  conf,
  numCores,
  maxMemory,
  storageMemory)
```

### 代码生成

Spark使用代码生成优化查询执行：

```scala
// 生成专用代码处理特定查询
val expressions = ctx.generateExpressions(
  expressions,
  inputTypes)
```

### 分区控制

Spark通过分区控制并行度：

```scala
// 控制分区数量
val rdd = sc.textFile("path", numPartitions)

// 重新分区
val repartitioned = rdd.repartition(newNumPartitions)
```
